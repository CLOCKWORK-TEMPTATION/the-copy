تحديث وإحكام الحزم (Dependencies)

متابعة ما بدأ في Worktree-2:

مراجعة الحزم التي تم الإشارة إليها كـ “قديمة/متداخلة” في تقاريرك (خاصة types، eslint، sentry).

تثبيت نسخ متوافقة تمامًا مع Node 20 و Next 15 و TypeScript 5.7.

تثبيت وثائق الإنتاج

إضافة FIXES_APPLIED.md الموجود في الأرشيف إلى المستودع فعليًا (مثلاً في الجذر أو تحت docs/).

إنشاء docs/PRODUCTION_READINESS_CHECKLIST.md يربط مباشرة بنقاط P0 + P1 لتكون لديك قائمة رسمية لكل نشر.

تهيئة المراقبة في البيئة الفعلية

تفعيل Sentry DSN الحقيقي للـ frontend والـ backend.

ربط /metrics بـ Prometheus أو أي APM آخر.

إعداد Alerting بسيط:

Error rate فوق threshold معيّن.

Latency متوسطة تتخطى X ms.

Bull queues بها عدد Jobs متراكم كبير أو failures متكررة.

تقوية إعدادات CORS والـ Security Headers في الإنتاج

في .env الخاص بالـ backend:

CORS_ORIGIN تضبط على domain واحد أو اثنين فقط (مثلاً https://thecopy.app).

مراجعة إعدادات Helmet لتأكد أنها لا تكسر أي resource خارجي ضروري في الـ frontend (خطوط، Sentry، إلخ).

P2 – تحسينات جودة وتجربة

رفع تغطية الاختبارات

Backend:

اختبارات لـ auth, projects, scenes, analysis controller باستخدام supertest.

Frontend:

Unit tests للمكوّنات الأساسية في Directors Studio (ProjectTabs, ProjectContent, ShotsPage, ScenesPage) + بعض integration tests.

تحسين الأداء بشكل مستمر

تشغيل npm run performance:report / budget:report في الـ frontend وتحسين النقاط التي تتجاوز الميزانيات.

تفعيل battery/perf detection للجسيمات (particles) على الأجهزة الضعيفة كما هو مخطط في سكربتاتك.
